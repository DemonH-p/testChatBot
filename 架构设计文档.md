## 一、系统概述

### 1.1 设计目标

设计一个**全双工语音对话系统 v3.2**，具备以下核心能力：

1. **实时语音采集与处理**：从麦克风采集音频流
2. **声学VAD**：使用WebRTC检测用户语音的起止与打断，阈值400ms
3. **Qwen ASR 17B**：使用Qwen ASR 17B模型将语音转换为文本，帧长20ms
4. **Qwen3 Omni Flash语义VAD**：流式判断语义完整性 + 有效人声判断
5. **并行情绪识别**：使用Qwen3 Omni Flash模型识别用户情绪
6. **融合阶段**：文本和情绪同时输入LLM进行任务规划
7. **意图理解与任务规划**：LLM理解用户意图并规划任务
8. **工具调用**：支持SKILLS和MCP协议的工具调用
9. **结果总结**：LLM对工具调用结果进行总结
10. **语音合成**：通过Edge TTS将文本转换为语音输出
11. **全双工交互**：语义VAD判断有效人声 → 立即停止TTS → 继续接收完整句子
12. **多轮对话**：支持上下文，对话连贯
13. **时延监控**：实时追踪各模块处理时延，便于性能分析

### 1.2 声学VAD与语义VAD分工

| 类型 | 职责 | 模型 |
|------|------|------|
| **声学VAD (aVAD)** | 听**声音有没有人在说话**：人声起点检测、静音检测、噪声过滤、触发打断确认 | WebRTC VAD |
| **语义VAD (sVAD)** | 听**话有没有说完**：语义完整性判断、有效人声判断、语气助词过滤 | Qwen3-Omni-Flash |

**全双工核心 = aVAD 快 + sVAD 准 + 可打断**

### 1.3 v3.2 核心改进

#### 打断逻辑优化（v3.2 最重要的改进）

```
┌─────────────────────────────────────────────────────────────────┐
│                    v3.2 打断流程（语义驱动）                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  TTS正在播报                                                    │
│       │                                                         │
│       ▼                                                         │
│  声学VAD检测到人声                                               │
│       │                                                         │
│       ▼                                                         │
│  启动ASR流式识别（打断确认模式）                                  │
│       │                                                         │
│       ▼                                                         │
│  语义VAD判断人声有效性                                           │
│       │                                                         │
│       ├──── 语气助词(嗯/啊/呃) ──→ 继续播报，继续等待              │
│       │                                                         │
│       ├──── 有效人声 ──→ 立即停止TTS播报                         │
│       │              继续接收用户输入                            │
│       │                                                         │
│       └──── 语义完整 ──→ 进入LLM处理                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**关键改进**：
- **不等声学VAD静音阈值**：语义VAD判断有效人声后立即停止TTS
- **区分语气助词**：避免"嗯"、"啊"等语气词误触发打断
- **继续接收输入**：停止TTS后继续接收用户完整句子，再进入LLM

## 五、打断机制详解

### 5.1 打断触发条件

v3.2 采用**语义驱动打断**，核心逻辑：

```python
def _check_interrupt_voice_validity(self, text: str) -> str:
    """
    检查打断确认结果

    Returns:
        "valid" - 确认是有效人声（应停止TTS）
        "complete" - 语义完整（应进入LLM处理）
        "pending" - 仍在判断中
        "timeout" - 超时
    """
```

### 5.3 打断流程代码

```python
async def process_audio(self, audio_chunk: bytes) -> Optional[DialogResult]:
    # 1. 打断确认模式下的处理
    if self._interrupt_confirm_mode and self._is_streaming:
        await self._process_audio_parallel(audio_chunk)

        interrupt_result = self._check_interrupt_voice_validity()

        if interrupt_result == "valid":
            # 确认是有效人声，立即停止TTS播报
            self._stop_tts_for_interrupt()

        elif interrupt_result == "complete":
            # 语义完整，结束打断确认，进入LLM处理
            return await self._finalize_interrupt_to_llm()

        elif interrupt_result == "timeout":
            # 超时处理
            ...

    # 2. SPEAKING状态下检测打断
    if self.dialog_state.state == DialogState.SPEAKING:
        if self.acoustic_vad.check_interrupt(audio_chunk):
            # 启动打断确认模式
            return await self._start_interrupt_confirmation()
```

## 十、总结

### 10.1 v3.2 核心特性

1. **语义驱动打断**：不等声学静音阈值，语义VAD判断有效人声后立即停止TTS
2. **区分语气助词**：避免"嗯"、"啊"等语气词误触发打断
3. **继续接收输入**：停止TTS后继续接收用户完整句子
4. **流式处理**：ASR流式输出 + 语义VAD流式判断
5. **并行处理**：语义VAD和情绪识别并行执行
6. **上下文连贯**：多轮对话支持上下文
7. **时延监控**：实时追踪各模块处理时延

### 10.2 关键优化

| 优化点 | 效果 |
|--------|------|
| 语义驱动打断 | 打断响应时间从>400ms降到<200ms |
| 语气助词过滤 | 避免误触发打断 |
| 流式ASR | 更快开始识别 |
| 流式语义VAD | 边说边判断，减少等待 |
| 并行处理 | 延迟降低 30-50% |
